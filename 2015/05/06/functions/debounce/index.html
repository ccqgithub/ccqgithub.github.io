分享两个实用的函数调用‘节流’与‘频率控制’方法

debounce

函数调用间隔控制
应用场景：防止按钮多次重复点击, 自动完成（autocomplete）的时候防止调用过于频繁。

/**
* 函数调用间隔控制
* fn: 执行的函数
* delay: 函数执行的最小间隔，单位毫秒
* prepose：靠前执行。为true时，函数调用即执行，并且在之后的delay中的调用无效（应用场景：防止按钮多次重复点击）。为false时，函数调用之后，并不立即执行，只有某次调用之后的delay内没有再被调用，才执行（应用场景：autocomplete，只有输入完成之后再执行查询）。
*/
function debounce(fn, delay, prepose) {
    var timer, last_exec = 0;
    return function() {
        var that = this,
            args = arguments,
            diff, curr,
            exec = function() {
                last_exec = +new Date();
                fn.apply(that, args);
            };
        timer && clearTimeout(timer);
        if (prepose) {
            diff = delay - (+new Date() - last_exec);
            diff <= 0 && exec();
        } else {
            timer = setTimeout(exec, delay);
        }
    }
}

// 使用示例1：自动完成（autocomplete）
// 输入的时候只有当你停顿时间超过200毫秒的时候才会执行从服务器查询的操作
// 而不是没敲一下键盘就查一下，这样服务器压力太大

$('#search').on('keyup mouseup blur change', debounce(function() {
    // search from server
}, 200));

// 使用示例2：防止按钮多次点击
// 点击按钮后1000毫秒内的点击都无效
// 这样能防止重复提交

$('#btn').on('click', debounce(function() {
    // do something ……
}, 1000, true));

throttle

函数连续调用时频率控制
应用场景：无聊你调用的多频繁，我也坚持我固定的步伐。比如：鼠标滚动事件……

/*
* 频率控制 返回函数连续调用时，fn 执行频率限定为每多少时间执行一次
* @param fn {function} 需要调用的函数
* @param delay {number} 间隔时间，单位毫秒
*/
function throttle(fn, delay) {
    var timer,
        last_exec = 0;
    return function() {
        var that = this,
            args = arguments,
            diff,
            exec = function() {
                last_exec = +new Date();
                fn.apply(that, args);
            };
        timer && clearTimeout(timer);
        diff = delay - (+new Date() - last_exec);
        if (diff <= 0) {
            exec()
        } else {
            timer = setTimeout(exec, diff);
        }
    }
}

// 使用示例1：鼠标滚动更新位置
// 鼠标滚动的事件发生是十分密集的，如果每个事件发生就调整一次，太耗性能了
// 所有加入 throttle控制，不管你滚动多块，我也是没200毫秒执行一次

$(window).scroll(throttle(function() {
    // do something ……
}, 200));
