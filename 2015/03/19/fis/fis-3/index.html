这一章，介绍一下FIS的配置，虽然fis文档很充分，不过我还是喜欢按我的理解整理一下。FIS相关文档见：FIS配置

fis-conf.js

项目根目录下的fis-conf.js是FIS配置的最顶端，这里的配置将会覆盖fis的其他相同的配置。

配置方法

// 获取配置
var conf = fis.config.get('project.fileType.text');

// 合并配置
fis.config.merge({
    project : {
        fileType : {
            text : 'tpl, js, css'
        }
    }
});

// 覆盖配置
fis.config.set({
    project : {
        fileType : {
            text : 'tpl, js, css'
        }
    }
});

变量配置

fis可以配置自己的变量，然后再配置’roadmap.path’是引用它们：

// 设置
fis.config.merge({'view': '/public/view'});
// 引用
fis.config.merge({
    roadmap: {
        path: [
            {
                reg : /^\/view\/(.*)$/i,
                useCache : false,
                release : '${view}/$1'
            }
        ]
    }
});

项目项目配置

  • charset
    指定项目编译后产出文件的编码，默认‘utf-8’
  • md5Length
    文件MD5戳长度，默认7
  • include
    设置项目源码文件include过滤器。只有命中include的文件才被视为源码，其他文件则忽略。可以是数组、字符串或者正则。不设置的话，表示项目下全部都是源码。
  • exclude
    设置项目源码文件exclude过滤器。可以是数组、字符串或者正则。如果同时设置了 project.include 和 project.exclude 则表示在include所命中的文件中排除掉某些文件。
  • fileType.text
    追加文本文件后缀列表,让fis系统编译是将这些后缀的文件当作文本文件处理，而不是二进制。
  • fileType.image
    追加图片类二进制文件后缀列表。让fis系统编译是将这些后缀的文件当作二进制文件处理。

插件配置

  • parser

    配置编译器插件，可以根据 文件后缀 将某种语言编译成标准的js、css、html语言。

对应fis的parse阶段的插件，比如：fis.config.merge(‘modules.parser.less’, ‘test’),表示less后缀的文件需要经过‘fis-parser-test’的插件的编译

其中‘modules.parser.xxx’配置文件对应的编译插件是什么，‘roadmap.ext.xxx’配置把那些后缀的文件当作’css、js、html’的哪一种处理。

‘modules.parser.xxx’ 和 ‘roadmap.ext.xxx’ 一般配合着编译插件一起使用。

  • preprocessor

    配置 标准化预处理器插件，可以根据 文件后缀 对文件进行预处理。

  • postprocessor

    在fis对js、css和类html文件进行语言能力扩展之后调用的插件配置，可以根据 文件后缀 对文件进行后处理。该阶段的插件可以获取文件对象的完整requires信息。

  • lint

    单文件编译过程中的代码检查插件。

  • test

    单文件编译过程中的自动测试插件。

  • optimizer

    单文件编译过程中的最后阶段，对文件进行优化。

  • prepackager

    打包预处理插件。

  • packager

    打包处理插件。

  • spriter

    打包后处理csssprite的插件。

  • postpackager

    打包后处理插件。

目录规范与域名配置

roadmap.path, 定制项目文件属性，包括但不限于 产出路径，访问url，资源id，默认依赖，文件类型。

  • 这里配置哪些路径下的哪些文件，该发布到哪里，该怎么处理，该携带些什么信息。

  • 注意：由于roadmap.path是一个数组，fis.config.merge会覆盖原有信息而不是合并，如需合并，可采用如下方式：

    // 为所有样式资源开启csssprites
fis.config.get('roadmap.path').unshift({   
    reg : /^\/src\/widget\/(.*)\.(css|less|styl)$/i,
    useSprite: true
});

  • 注意：roadmap.path是按数组的顺序一个个匹配，一旦匹配就不再进行匹配，这意味着几个配置包含了相同的文件，后面的配置中该文件将被忽略。

  • reg: ‘**’, 表示所有文件。
  • ‘map.json’，是fis打包生成的资源文件，你也可以给它指定发布路径

部署配置

设置项目的发布方式。

  • 部署配置可以设置多个，可以是本地文件夹，也可以是远程服务器。可以为不同的文件指定任何不同的发布路径，fis照样正常能胜任。
  • 每个部署配置可以设置 字符串替换

打包配置

配置要打包合并的文件。

//fis-conf.js
fis.config.merge({
    pack : {
        //打包所有的demo.js, script.js文件
        //将内容输出为static/pkg/aio.js文件
        'pkg/aio.js' : ['**/demo.js', /\/script\.js$/i],
        //打包所有的css文件
        //将内容输出为static/pkg/aio.css文件
        'pkg/aio.css' : '**.css'
    }
});
