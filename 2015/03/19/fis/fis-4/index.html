几个重要插件

fis-postprocessor-jswrapper

它能将按照node.js写法的模块封装成cmd模块

// 源代码
module.exports = 'test';

// 封装后
define('id', function(require, exports, module) {
    module.exports = 'test';
});

// 也可以自己定义封装模板
fis.config.merge({
    settings: {
        postprocessor: {
            jswrapper: {
                type: 'amd',
                template: 'define(function(require, exports, module) {${content}});'
            }
        }
    }
});

// 封装后 就没有了id，这样就能让后面的amd插件实用
define('id', function(require, exports, module) {
    module.exports = 'test';
});

fis-postpackager-autoload

它能将当前页面的所有资源依赖自动注入页面中，由于fis的打包过程，只是将资源依赖存储于一个map.json中，并不插入页面（它这样做有它很重要的道理）。如果我们不想再页面展示后才根据依赖再去一个个载入文件，就要用它了。

fis-postpackager-simple。

用于自动打包页面零散资源和应用打包资源的FIS插件。

  • 自动将页面中声明的资源引用替换为pack中设置的资源
  • 自动将未打包的零散资源按照引用顺序打包，默认关闭

fis-packager-depscombine。

另一种打包方案，在 pack 中命中的文件，其依赖也会自动命中。如果你采用的是手动配置打包，建议使用此插件来简化配置。

fis-postprocessor-amd。

fis amd 支持，完全满足 amdjs 规范。
